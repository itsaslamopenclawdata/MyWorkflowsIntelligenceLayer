Weekly AI drop roundup from Andrew Warner + Corey Ganim covering: (1) Codex Record & Replay — record yourself doing a task once, turn it into a reusable skill in one shot; (2) OpenRouter savings; (3) AEO (Answer Engine Optimization) score tools; (4) Claude in Slack via Claude Tag (Anthropic's enterprise/team feature); (5) Zapier MCP's 5-minute setup for granular Google Workspace permissions; (6) Bluerails / Agent Mail / Blue-Rails for making your site transactable by AI agents; (7) Claude Code Artifacts (the in-IDE artifact pane that mirrors Claude chat's artifact UX).

## Key Insights

- **Codex Record & Replay is the "long time coming" skill primitive.** Andrew demos going into Codex, saying "record and replay," uploading a YouTube video, doing the upload task naturally once, and ending with a reusable Codex skill. Corey's old workflow: record with Loom → upload to Gemini (only model that watches video) → get step-by-step checklist → copy into Claude → turn into skill → iterate 2–5 times. New workflow: one-shot. "One of the most impactful updates we've seen in a long time." (0:30–2:20)

- **AEO score tools exist and your "Andrew Warner" might come back as a poker player.** Type your name into a Hacker News-popular AEO tool: it tells you which models describe you correctly vs. as the wrong person entirely (Andrew's result: an American poker player). Output also shows the top-3% strength score + model-by-model fact-check. Takeaway: if your personal brand matters, run this and fix the obvious errors across the top-10 models. (1:45–2:50)

- **Claude Tag (Anthropic) puts Claude in your Slack.** The reference is to a Claude Code/Slack integration that lets you @mention Claude in a Slack channel and get responses in-thread. Only available on Enterprise + Team accounts, which is the right account tier for teams anyway. (5:35–6:05)

- **Zapier MCP: the 5-minute Google account story.** Andrew tried Google's native tool to give his Hermes agent its own Google account (email, calendar, Drive) — failed for an hour, even with the Googler who built it. Then he went to zapier.com/mcp, selected checkboxes for which tools the agent gets, and set per-account permissions: agent's email can send/receive, your email can only read/draft. Done in minutes. The pattern: agentic MCP setups are converging on Zapier as the lowest-friction orchestrator. (6:00–7:15)

- **Bluerails is the "PayPal button for AI agents."** Two-part service: (1) make your site crawlable / readable / transactable by agents (so Claude/ChatGPT web fetch actually works on it), (2) add a checkout button for AI agents so they can buy on behalf of users. Andrew's take: 6–12 months too early but claim market share now because the wave is coming. Corey's broader frame: "now is the time for hustler go-getters to make their sites more accessible to agents." (7:10–8:45)

- **Agent Mail: Gmail for your agent, no setup.** Tell your agent "create an account," it does it; you get a single-click human-confirmation email. It's not a better product than Gmail — Andrew is messaging the founder with feedback — but he uses it *because* of the frictionless onboarding. The lesson: onboarding friction beats product quality for agents; whoever removes the OAuth dance wins. (8:45–9:25)

- **Claude Code Artifacts closes the gap with Claude chat.** Previously, the "artifacts on the right" UX only existed in Claude chat / Cowork; Claude Code was chat-only. Now Claude Code has the same artifact pane — shareable, updates in real time. Eric (sitting in): "this is the one thing that Claude Code didn't have that Claude chat and Cowork did." (9:40–10:30)
