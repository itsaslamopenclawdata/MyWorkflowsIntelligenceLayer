# "Learn AI" Is Bad Advice. Learn This Instead

**Channel:** Greg Isenberg  **Published:** 2026-06-25  **Duration:** 29:53  **Watch:** https://www.youtube.com/watch?v=jJIOA4GroBw

## TL;DR

Solo episode framing six "AI-proof" skills Greg argues will *increase* in value as models improve — none require credentials, all can be started this weekend: (1) agents + local AI, (2) distribution, (3) robotics + hardware + sourcing, (4) curation / "yapping", (5) the builder-distributor (ship + sell solo), (6) IRL community building. Each skill comes with one concrete "first rep" exercise. The thesis: shipping one of these won't make you the AI-native founder, but combining three makes you the person every team wants in the room.

## Key Insights

- **Agent orchestration is the grown-up version of prompt engineering.** Greg defines the bar as designing "an AI employee that has contacts, has tools, has permissions, has memory, has a goal, and a way to check its own work before it bothers you." The first rep: build a daily briefing agent wired to your calendar + notes + saved links, with a hard rule that it must show sources and request approval before sending anything. (0:57–4:40)

- **Distribution ≠ posting.** In a world where anyone can ship a landing page, the bottleneck moves to "can you make people care?" Greg frames modern marketers as part researcher / part storyteller / part media operator / part community builder. First rep: do a distribution map — write down the 20 places your niche's attention already lives, find one painful sentence they'd actually say out loud, then write 20 hooks (curiosity / fear / status / money / "wish I knew earlier"). (4:52–9:00)

- **Robotics is the new SaaS.** "The last decade rewarded people who moved pixels. The next decade rewards people who can move atoms too." The cost barrier is gone: open-source robot learning (Hugging Face Le Robot), cheap arms (SO-100/SO-101 ecosystem), small VLA (vision-language-action) models, community datasets. First rep: buy/assemble a low-cost arm, attach a cheap camera, teach it one boring task (sort 3 objects, press a button), then *document every failure mode*. (9:03–14:20)

- **Curation is the post-AI-slop moat.** "Algorithms are prioritizing yapping because they are seeing AI slop move into the timeline and people are getting tired of that." First rep: 7-day curation sprint in a single niche — every day, find 3 things and post one short video using the fixed structure "I saw this. Most people think it means this. I think it actually means this. Here's the move." Forces a take. (14:28–19:00)

- **The builder-distributor compresses the Wozniak/Jobs split.** AI lets one person prototype, copy-write, shoot demo video, DM first 100 users, edit clips, and iterate. First rep: a 48-hour loop — pick one tiny problem you personally understand, build the smallest version with AI, then create 10 pieces of distribution before you feel ready (1 demo, 3 clips, 3 posts, 2 DMs, 1 landing page). Greg references his ACP-funnel framework here: audience → community → product. (19:08–23:10)

- **IRL community is the only thing AI makes scarcer.** "AI makes content abundant. Software abundant. Advice abundant. So scarcity moves toward belonging, trust, and context." First rep: don't start with a massive event — host 6–8 people around one sharp question (dinner / walk / hike / breakfast), then send a short recap with the best quotes, inside jokes, and one follow-up for everyone. The recap is what turns the room into a network. (23:10–27:00)

- **The skill stack is the actual moat.** "Pick one and get dangerous. Pick two and you have some leverage. Pick three and you become the kind of person that everyone wants on the team, in the room, or building the company." (27:27–29:53)
