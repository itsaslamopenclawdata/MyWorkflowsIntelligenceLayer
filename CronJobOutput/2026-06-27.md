## 6. Failure Modes and Auto-Recovery

- **Failure:** Google Maps scraper returns 0 results or is rate-limited (HTTP 429).
  - **Detection:** `scrape_dental_offices.py` exits with code 0 but CSV has <10 rows, or HTTP 429 logged.
  - **Auto-recovery:** Subagent 2 falls back to scraping LinkedIn Sales Navigator via the same scrape_honey_contacts.py pattern (already supports LinkedIn source), OR uses the existing `DentalRealData/DentalRealData-2026-05-19.md` research notes as a seed list (city/state already identified). If both fail, subagent 2 generates a synthetic prospect list from public dental directories (Yellow Pages, Yelp dental category).
  - **Escalate to Aslam only if:** 0 dental prospects are available after 30 min of retry.

- **Failure:** SMTP env vars unset (cold-outbound-sender.py exits with `KeyError: 'SMTP_HOST'`).
  - **Detection:** Step 6 command fails with non-zero exit code; error message contains "SMTP_HOST" or "SMTP_PASSWORD".
  - **Auto-recovery:** Fall back to `--send-method cli` with `gog gmail send` (already supported in script defaults). The `gog` CLI uses Aslam's pre-authorized Google credentials (already wired). No Aslam intervention needed.
  - **Escalate to Aslam only if:** Both SMTP and `gog gmail` CLI fail. Then Aslam pastes SMTP creds in a 5-min Telegram exchange.

- **Failure:** Subagent 1's `dental_reactivation_stack.py` has a bug (ImportError on the verify check).
  - **Detection:** Step 2's `python3 -c "from scripts.dental_reactivation_stack import ..."` returns ImportError.
  - **Auto-recovery:** Subagent 1 re-runs with the specific error message attached to the goal string. 2 retries max.
  - **Escalate to Aslam only if:** 3 consecutive failures with the same error.

- **Failure:** No positive replies received within 3 days of the cold send.
  - **Detection:** `data/cold-outbound-replies.json` is empty after Day 3.
  - **Auto-recovery:** Subagent 4 sends a Day-3 follow-up email (Hamza's "long-tail nurture" pattern from chapter 5:40-7:40) using a shorter, curiosity-driven subject: `Subject: Re: {original subject} — 1 question`. Body: "Hi {Name}, just bumping this in case it got buried. Any interest in a free lapsed-patient audit for {Practice Name}? — {Aslam's First Name}".
  - **Escalate to Aslam only if:** Zero replies after Day 3 follow-up + Day 7 follow-up (kill condition: pivot to next vertical or next playbook).

- **Failure:** Stripe Payment Link creation fails (Stripe API key unwired).
  - **Detection:** `stripe.Invoice.create(...)` returns 401 Unauthorized.
  - **Auto-recovery:** Subagent falls back to PayPal Invoice (`paypalrestsdk` API call). If PayPal also unwired, generates a manual invoice template at `dental-reactivation/audits/{slug}-invoice.txt` for Aslam to send via email.
  - **Escalate to Aslam only if:** Both Stripe and PayPal APIs fail AND Aslam must create a Stripe Payment Link manually (5 min).

- **Failure:** HIPAA concern — the cold email includes patient data that constitutes PHI.
  - **Detection:** Subagent 3's email body accidentally includes any actual patient name or appointment date from a real PMS.
  - **Auto-recovery:** Subagent 3 regenerates the email using only synthetic placeholder names ("Maria", "John") and zero real patient data. The cold email is always sent to the office manager, not to patients directly — patient outreach happens only after the office manager signs the BAA.
  - **Escalate to Aslam only if:** A real PMS is connected during the pilot and a BAA must be signed before any patient outreach.
