## 6. Failure Modes and Auto-Recovery

Top 3 likely failures:

- **Failure:** GitHub API rate-limit (60/hr unauthenticated) hits when `github_publisher.py` tries to push to `itsaslamopenclawdata/GumRoadGeneratedPDFs`.
  - **Detection:** `build_product.py` exit code non-zero AND stderr contains `403` or `rate limit` or `API rate limit exceeded`.
  - **Auto-recovery:** Re-run with `--no-push` flag (the 3 files stay local at `/tmp/GumRoadGeneratedPDFs/15_Claude_Code_Rules_Pack/`); Gumroad still publishes via `gumroad_publisher.py`'s Playwright flow even if the GitHub push failed. The GitHub push is a courtesy landing-page proof, not a hard gate for Gumroad sales.
  - **Escalate to Aslam only if:** The Gumroad publisher itself fails (Steps 4-6) — i.e. Playwright can't log in or form-fill fails. Aslam may need to manually paste the title/desc into Gumroad.com.

- **Failure:** The transcript is too thin for the LLM-driven content generator to produce a 12+ page PDF (rare for this transcript; it's 5KB+ with 15 distinct rules).
  - **Detection:** `verifiers.py` returns `pages: <12` OR `STATUS: FAIL`.
  - **Auto-recovery:** Re-run build_product.py with `--type framework` instead of `--type checklist`. The framework topic-type forces more in-depth narrative around each rule, which usually pushes page count past 12. If still failing, fall back to the manual-content-extraction pattern: read the transcript, write a content_dict JSON to `/tmp/15_claude_code_rules_content.json` matching the schema in `pdf_generator.py` docstring, and call `pdf_generator.py` directly with `--content-file`.
  - **Escalate to Aslam only if:** All three auto-recovery paths fail AND the LLM helper (`/tmp/llm_call.py`) is broken. Then a fresh Hermes session with manual content-authoring is needed.

- **Failure:** Aslam doesn't tap APPROVE on Telegram within 24 hours.
  - **Detection:** `gumroad_publisher.py --approve` logs the timeout; no Telegram callback received within `APPROVE_TIMEOUT` env var (default 86400s = 24h).
  - **Auto-recovery:** Cron-run the next day detects the orphan draft, posts a follow-up Telegram reminder with the same draft content. If still no response after 48h, mark the workflow as `STALLED — needs Aslam APPROVE` in the outcome doc and queue a re-prompt.
  - **Escalate to Aslam only if:** Aslam taps CANCEL or EDIT. Then re-build the description per Aslam's edit instruction and re-post for APPROVE.
