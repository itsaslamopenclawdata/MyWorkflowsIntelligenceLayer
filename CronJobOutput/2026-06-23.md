## 6. Failure Modes and Auto-Recovery

- **Failure:** `build_product.py` crashes because `weasyprint` system libraries are missing (libpango, libcairo).
  - **Detection:** Exit code 1, stderr contains "OSError: cannot load library".
  - **Auto-recovery:** Subagent `weasyprint-fixer` goal = "Install missing weasyprint system libraries on this WSL box. Run `sudo apt-get update && sudo apt-get install -y libpango-1.0-0 libpangoft2-1.0-0 libcairo2`. Then re-run Step 2." toolsets = [terminal]. Re-runs Step 2 after install. Escalate to Aslam only if `sudo` fails (means system policy blocks installs).

- **Failure:** GitHub push fails (creds expired, branch protection, or rate limit).
  - **Detection:** `git push` stderr contains "Authentication failed" or "403 rate limit"; `git ls-tree origin/main GumRoadGeneratedPDFs/sell-to-ai-agents/` returns empty.
  - **Auto-recovery:** Subagent `git-push-fixer` goal = "Re-auth git using ~/.git-credentials, force-push the single new folder GumRoadGeneratedPDFs/sell-to-ai-agents/ to origin main, return success/failure." toolsets = [terminal]. If still failing after one retry, skip the push — keep the local files in `/tmp/GumRoadGeneratedPDFs/sell-to-ai-agents/` and ship them to Gumroad directly via `agent-reach` (Step 5 still works with local files). Escalate to Aslam only if no path forward after two retries.

- **Failure:** `verifiers.py` rejects the PDF (page count < 12, or output folder has extra files).
  - **Detection:** `build_product.py` prints `QA: FAIL ...` and exits non-zero.
  - **Auto-recovery:** Subagent `pdf-debugger` goal = "Read `verifiers.py` lines 1-172 and the last 50 lines of `build_product.py` output. Diagnose why the QA gate failed. Re-run content_generator.py with the topic only (no overrides). Re-render the PDF. Return the fix and a re-verified PASS." toolsets = [terminal, file]. Escalate to Aslam only if the topic-specific content generation is genuinely broken (rare — the template fallback always yields 12+ pages).

- **Failure:** Gumroad subagent can't log in (session cookie expired).
  - **Detection:** `agent-reach` browser returns login page or 401.
  - **Auto-recovery:** Subagent `gumroad-login` goal = "Re-login to Gumroad seller dashboard using the credentials stored in ~/.config/hermes/gumroad.env. Save a fresh session cookie. Hand off to subagent 3 (gumroad-lister)." toolsets = [terminal, web]. Escalate to Aslam only if the env file is missing entirely.
