## 6. Failure Modes and Auto-Recovery

Top 5 likely failures:

- **Failure: Transcript file is missing or the channel-pipeline has rolled it back.**
  - **Detection:** Step 1's `ls` returns empty.
  - **Auto-recovery:** Hermes falls back to the compiled note file `2026-06-03_7-point-niche-scoring-framework.md` (which already contains the 7 criteria summary in the "Key Insights" section) and re-extracts from there. The note's transcribed-status is "full" so this is safe.
  - **Escalate to Aslam only if:** both transcript AND note are missing (would require the whole channels/vendasta/ folder to be gone, which is a deeper data-loss event).

- **Failure: Gumroad login fails (`gumroad_publisher.py --test` returns non-zero, or `--request` says "Login failed").**
  - **Detection:** `gumroad_approval_state.json` shows `pending=true` for >5 minutes with no follow-up Telegram message.
