# 7 Hermes uses you must try (that actually work)

**Channel:** The Next New Thing  **Published:** 2026-06-24  **Duration:** 37:57  **Watch:** https://www.youtube.com/watch?v=6BHh8BoKim4

## TL;DR

Andrew Warner + Eric Siu walk through seven concrete Hermes Agent use cases from real entrepreneurs: (1) desktop control of VS Code/Claude Code overnight, (2) recurring competitor research turned into reusable markdown, (3) a "resolver" agent that watches your screen and skillifies what you do, (4) a daily memory wiki via Hermes + Obsidian + QMD, (5) Zapier MCP for granular agent access to Google accounts, (6) the Slack-vs-Discord-for-AI-agents tradeoff, (7) Hermes running creative/manim skills to produce videos. Vendor-disambiguation note: the Hermes Agent discussed here is the **third-party personal AI agent platform** referenced by Greg Isenberg and others — it is **not** MiniMax Hermes Agent.

## Key Insights

- **Use case 1 — Desktop control of your coding agent.** Andrew demos telling Hermes "go to my computer, open VS Code, and in Claude Code plugin type this URL. Do not stop until you copy their whole business." His framing: he has unused Claude credits overnight and wants Hermes to drive Claude Code in a loop until a target is hit. The killer combo: Hermes as the orchestrator + Loop/Claude Code as the executor. (0:00–1:40)

- **Use case 2 — Recurring competitor research → markdown → Claude Code.** Alex's pattern: prompt Hermes to "do a full research and technical breakdown of the site. What stack does it use? How does it work? Which features does it have? What can we emulate?" Then dump the result to markdown and hand it to Claude Code or Codex to clone the chosen features. Eric's caveat: one-offs are useless; the value is when it's a daily/weekly/monthly cron that builds a corpus over time. Many Hermes tasks "go into the ether" if not turned into a recurring routine. (1:45–3:30)

- **Use case 3 — The "resolver" agent (skillify yourself).** Eric built a tool that watches what he's doing in Claude Code / Codex / Hermes all day, then at the end says "here are the things you should loop. These are the things you should continue." It then auto-suggests the loop/automation. He named it himself ("resolver"). Takeaway: instead of manually building automations, instrument an observer and let the agent propose what to skillify. (5:40–7:20)

- **Use case 4 — Daily memory wiki (Hermes + QMD + Obsidian).** Eric's stack: QMD (Toby Luke's memory tool — literally just the letters "QMD" on GitHub) makes Hermes "less dumb and easier to find things"; Obsidian holds the daily log of decisions; Linear holds collaborative projects. Andrew pushes grocery-list-to-Hermes-via-Telegram as the simplest example of "offload friction to the agent." (7:55–9:50)

- **Use case 5 — GBrain / GStack as the "skills catalog."** Eric explicitly says GBrain is "not a memory thing — it's a list of really good skills you can use: CEO review, design review, things like that." This is the gstack framework Andrew's channel has covered separately. Use it when you want battle-tested prompts, not when you want Hermes to remember what you did last week. (10:00–11:10)

- **Use case 6 — Zapier MCP for granular agent access to Google.** Andrew tried Google's native tool to give Hermes agent its own Google account — spent an hour, still couldn't get it working even with a Googler's help. Then he switched to Zapier MCP at zapier.com/mcp: select checkboxes for which tools the agent gets, then granular per-account permissions (e.g., agent's email can send/receive, your email can only read/draft). The killer use case he actually uses: "Every day at 5pm, go to my YouTube channel, see who I published with, look up their email address from my sent folder, draft a thank-you message that I just hit send on." (12:30–14:00)

- **Use case 7 — Discord vs Slack for AI-agent-driven workflows.** Brian Castle argues Slack threading/multi-channel isn't reliable for agents; he moved to Discord. Eric disagrees for multiplayer — his Hermes rates Slack 9/10 because of connectors, multiplayer, and team availability. Andrew's heuristic: single-player → Discord, multiplayer (with team) → Slack. Don't try to drag the team to a new platform just to make your agent happy. (14:00–16:30)

- **Bonus — Hermes running creative skills (manim/hyperframes).** Eric had Hermes use the `creative` skill and `manm_video` (manim) skill to produce a video about how Hermes works, by chaining planning + code generation + asset search. Demonstrates skill composition: a meta-skill that calls lower-level skills. (16:30–17:30)
