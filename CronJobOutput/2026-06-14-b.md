### Step 2 — Score-filter the 100-row CSV to the top 10 by gap-clarity (Hermes, ~5 min)

- **WHAT:** From the 90 no-website rows, pick the 10 highest-fit companies for the mockup factory. Scoring formula (per the source playbook filter): 5+ years on the map = 3 pts (we can't get "years on the map" from this CSV, so use a proxy: company name length + multiple-word name + non-po-box address → 3 pts); under 50 reviews = 1 pt (proxy: presence of a Google-search hint; not available in this CSV, so all rows get 1 pt as base); no website = 4 pts (the dominant signal — every row that lacks `website` gets the 4 pts); solid offline footprint proxy: state-cell present + phone present + city-cell is non-empty + 3-word-plus name → 2 pts. Total possible: 10. Pick the top 10 by score, with state-diversity tie-break (do not pick 5 from the same state — keep the geographic spread that makes the campaign look intentional).
- **WHO:** Hermes.
- **HOW:**
  ```bash
  /usr/bin/python3 <<'PY'
  import csv, re
  rows = list(csv.DictReader(open('/home/itsshaikaslam/MyHermesResearcher/honey_companies.csv')))
  def score(r):
      s = 0
      if not r['website'].strip(): s += 4
      if r['phone'].strip(): s += 1
      if r['address'].strip(): s += 1
      if r['state'].strip(): s += 1
      if len(r['name'].split()) >= 2: s += 1
      if len(r['name']) >= 12: s += 1
      if re.search(r'\b(Farm|Honey|Apiary|Bees|Bee)\b', r['name'], re.I): s += 1
      return s
  scored = [(score(r), r['state'], i, r) for i, r in enumerate(rows)]
  scored.sort(key=lambda x: (-x[0], x[1], x[2]))
  picked = []
  per_state = {}
  for sc, st, idx, r in scored:
      if len(picked) >= 10: break
      if per_state.get(st, 0) >= 2: continue
      picked.append((sc, st, idx, r))
      per_state[st] = per_state.get(st, 0) + 1
  print('TOP 10 PICKS:')
  for sc, st, idx, r in picked:
      print(f"  {r['name']:35s} | {r['state']:15s} | phone={r['phone']:18s} | website={r['website'] or '(none)':30s} | score={sc}")
  PY
  ```

## 6. Failure Modes and Auto-Recovery

Top 3 likely failures:

- **Failure:** SMTP credentials rejected by Gmail/Outlook (535 auth failure) at Step 8.
  - **Detection:** `cold-outbound-sender.py` returns non-zero exit + a `smtplib.SMTPAuthenticationError` traceback in stderr.
  - **Auto-recovery:** Hermes retries with `--send-method smtp --max 0` (dry-run-only) and falls back to writing 7 `.eml` files to `LocalBizMockups/_outbox/` for Aslam to forward manually from his mail client (60 sec/email). Aslam gets a Telegram alert: "SMTP rejected; 7 .eml files saved to LocalBizMockups/_outbox/. Forward them when you have 5 min."
  - **Escalate to Aslam only if:** Aslam's mail client cannot load the `.eml` files OR the prospect's reply is a hard bounce.

- **Failure:** GitHub Pages not enabled → mockup URL renders as a GitHub folder-view, not a styled page, on the prospect's first click.
  - **Detection:** Aslam opens one of the URLs in a browser and sees a directory listing.
  - **Auto-recovery:** Hermes switches the email mockup URL to the raw.githubusercontent.com form (`https://raw.githubusercontent.com/itsaslamopenclawdata/MyWorkflowsIntelligenceLayer/main/LocalBizMockups/<slug>/index.html`) which DOES render the page in any browser (raw GitHub serves HTML as `text/html` for direct navigation). Hermes re-sends the 7 emails with the updated URL. The raw URL is the production link for the entire campaign.
  - **Escalate to Aslam only if:** the prospect reports the page doesn't load (firewall blocks raw.githubusercontent.com — rare in US SMB).

- **Failure:** Step 3 brief.md generation produces 70-word email bodies outside the 60-80 word range, or with forbidden tokens ("AI", "Claude", "Lovable", "Higgsfield", "Hermes", "automation", "agent").
  - **Detection:** The Step 3 verification check fails: `forbidden_token_check` flags any of the 7 emails.
  - **Auto-recovery:** Hermes re-generates the offending brief.md's email field with a fresh template that respects the word count and the forbidden-token list. Runs the same verification check again. Re-iterates up to 3 times.
  - **Escalate to Aslam only if:** 3 iterations all fail — Aslam reads the candidate email and approves one.
